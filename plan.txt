Context

 The current improvements.md is organized as an engineer's technical backlog (storage migration first, features
 later). It's missing the core features that make Sunsama addictive — the daily rituals, timeboxing, capacity
 planning, and shutdown flows. The phase ordering is wrong for the user's goal: an app they open every morning, use
 throughout the day, and close with a shutdown ritual every evening. The user has no structured planning system today
  — this needs to build the habit.

 The user wants:
 - Morning: Guided daily planning ritual (Sunsama-style) + optional journaling
 - Time: Light timeboxing (estimate durations to plan the day, no active focus timer needed initially)
 - End of day: Guided shutdown ritual with auto-rollover fallback if skipped
 - Scope: Work-first, life second (work/personal contexts nice-to-have)
 - Integrations: Microsoft ecosystem + GitHub/Jira (eventually, not blocking)
 - Build order: Features track + Electron/SQLite track in parallel (separate agents)

 Revised Phase Structure

 NEW Phase 1 — "The Daily Habit" (make it worth opening every day)

 Goal: Create the core daily loop that builds the habit

 1. "My Day" as the default view — When you open the app, you see TODAY. Not a scrolling week view. Today's tasks in
 a focused single-column list, today's calendar events alongside, a capacity indicator showing planned vs available
 hours. The week/month view becomes a secondary navigation mode.
 2. Task time estimation — Add estimated_minutes: number | null to the Task interface. When creating or editing a
 task, optionally set a duration (15m, 30m, 1h, 2h, 4h as quick-pick buttons). Day columns and My Day view show total
  planned time. This is what makes planning feel like planning.
 3. Daily Planning Ritual (morning flow) — Triggered on first open of each day (or manually via a "Plan My Day"
 button). A guided modal/flow:
   - Step 1: "Yesterday's unfinished tasks" — show incomplete tasks from yesterday. For each: drag to today, push to
 tomorrow, send to backlog, or mark done.
   - Step 2: "Today's calendar" — show today's meetings/events (mocked initially, real later). Shows how much free
 time you have.
   - Step 3: "Pull from backlog" — browse backlog and drag tasks into today.
   - Step 4: "Capacity check" — "You've planned 6h 30m with 5h free after meetings. Consider trimming." with a visual
  bar.
   - Step 5 (optional): "Morning intention" — a text field for a daily journal/intention entry.
   - Step 6: "Ready to go" — confirms the plan, closes the modal, shows My Day view.
 4. Daily Shutdown Ritual (evening flow) — Triggered manually (button) or at a configurable time (notification if
 Electron). A guided modal:
   - Step 1: "Today's review" — show all tasks with checkboxes. Mark done/undone.
   - Step 2: "Incomplete tasks" — for each unfinished task: push to tomorrow, send to backlog, reschedule to specific
  date, or delete.
   - Step 3 (optional): "Reflection" — "How was today?" text field. Saved as a daily journal entry.
   - Step 4: "Day complete" — shows summary (X tasks done, Y pushed, Z hours planned vs actual).
 5. Auto-rollover fallback — If the user skips the shutdown ritual, unfinished tasks from yesterday automatically
 surface in the next morning's planning ritual (Step 1). No tasks silently vanish.
 6. Priority levels — Keep existing spec from improvements.md (urgent/high/medium/low/none with Sunsama-inspired
 visual design). This is essential for the morning planning flow to feel useful.

 Design Direction (expanded)

 - Inspiration: Sunsama — calm, intentional, ritual-driven. The app should feel like a personal assistant that gently
  guides you through your day, not a project management tool.
 - Morning ritual: Warm, inviting, step-by-step. Soft transitions between steps. Not a modal that feels like a chore.
 - Shutdown ritual: Reflective, calming. End-of-day energy. Summary feels like an accomplishment.
 - Capacity bar: Green when under-planned, amber when at capacity, red when over-committed. Always visible in My Day
 view.
 - Time chips: Small, rounded badges on task cards showing estimated duration (e.g., "1h", "30m"). Subtle but
 informative.
 - Font: Inter (keep). Dark mode: Keep class-based HSL strategy.
